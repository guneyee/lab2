# Epic Template

## 1. Epic Title
Multi-Project Prioritization View

## 2. Description (2-3 sentences)
Support project labels and clear prioritization within To Do so a solo developer can manage tasks across 2-3 concurrent projects. Keep prioritization lightweight with fast reorder and visible project context on each task card. This epic improves daily planning clarity and reduces time spent deciding what to do next.

## 3. Primary Persona (Who Benefits Most)
- Persona: Alex (Indie Builder)
- Why this persona: Alex balances feature, bug, and experiment tasks and needs clear cross-project prioritization.

## 4. Success Criteria (Measurable Outcomes)
- Cross-Project Clarity - Target: 100% of tasks can display an associated project label when provided.
- Prioritization Efficiency - Target: Daily planning median <=5 minutes within 2 weeks.
- PRD Metric Mapping - Target: Supports Metric 1 (Daily Planning Time Reduction).

## 5. Scope/Complexity (S/M/L Estimate)
- Size Estimate: S
- Scope Notes: Includes label capture/display and To Do ordering refinements; excludes advanced filters, analytics, and reporting.

## 6. Dependencies (What Must Exist First)
- EPIC-01 task schema with optional project label support.
- Board rendering support for metadata display on task cards.
- Persisted ordering model for To Do prioritization.
